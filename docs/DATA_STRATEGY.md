# Data Strategy for City1 v2

## Главная проблема

У нас нет надёжного target по населению на уровне квадратов сетки.

Поэтому нельзя честно утверждать, что мы обучаем классическую supervised-модель на "истинном населении ячейки".

## Что можно использовать

### 1. Официальные city-level totals

Подходящие источники:

- Бюро национальной статистики
- eGov / связанные государственные публикации
- официальные городские/региональные статистические сводки

### 2. Пространственные proxy-признаки

- здания
- этажность
- land use
- дороги
- остановки
- школы
- больницы
- магазины
- парки

## Как использовать city-level totals правильно

Официальная численность города полезна в трёх ролях:

1. `Calibration`
   - сумма прогнозов по ячейкам должна совпасть с официальной численностью города

2. `Constraint`
   - модель должна строить внутреннее распределение, но не менять городской total произвольно

3. `Weak supervision`
   - если есть несколько городов, можно обучать относительное распределение и калибровать его по официальным totals

## Чего city-level totals не дают

- они не говорят, сколько людей живёт в конкретной ячейке
- они не заменяют перепись по микрорайонам
- они не позволяют честно валидировать точность на уровне сетки

## Рекомендуемая научная формулировка

В v2 задачу лучше описывать так:

`Оценка пространственного распределения населения внутри города по геопространственным proxy-признакам с калибровкой по официальной численности населения города.`

## Минимально честная схема обучения

1. Собираем пространственные признаки по сетке.
2. Строим исходный latent score плотности населения.
3. Нормируем score так, чтобы сумма по ячейкам совпала с официальным total города.
4. Сравниваем между городами и по holdout-городам.

## Что желательно найти в будущем

Если получится, нужно искать частичные spatial labels:

- население по административным районам
- жилищный фонд по кварталам
- избирательные участки / домохозяйства / адресные агрегаты
- census-like spatial layers

Это позволит перейти от weak supervision к semi-supervised или supervised обучению.
